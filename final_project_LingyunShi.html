Data characteristics and predictors introduction¶

In total, there are 75,450 breath records, each lasts for around 3 seconds (can see from the time_step, which is the relative time to a start of a breath); Predictors include:

  1. R: lung attribute
  2. C: lung attribute
  3. u_in: the pressure of inspiratory solenoid valve; Ranges from 0 to 100
  4. u_out: the open status of the exploratory solenoid valve; Either 0 or 1
  5. pressure: pressure measurements in previous time steps

The outcome to predict is pressure: the airway pressure measured in the respiratory circuit, measured in cmH2O.

Machine Learning¶

The whole dataset was randomly split as training+development (70%) and testing (30%). The training and development datasets were used to develop traditional machine learning (regression-based) and deep learning models and tune hyperparameters for the developed models. The testing dataset was hold out for evaluating and reporting the model performance.

Evaluation¶

The study reported the following 5 evaluation metrics: MSE: mean squared error MAE: mean absolute error RMSE: root mean squared error MAPE: mean absolute percentage error R2: coefficient of determination

# Evaluation functions
from sklearn import metrics
import numpy as np
def timeseries_evaluation_metrics_func(y_true, y_pred):
    
    def mean_absolute_percentage_error(y_true, y_pred): 
        y_true, y_pred = np.array(y_true), np.array(y_pred)
        return np.mean(np.abs((y_true - y_pred) / y_true)) * 100
    print('Evaluation metric results:-')
    print(f'MSE is : {metrics.mean_squared_error(y_true, y_pred)}')
    print(f'MAE is : {metrics.mean_absolute_error(y_true, y_pred)}')
    print(f'RMSE is : {np.sqrt(metrics.mean_squared_error(y_true, y_pred))}')
    print(f'MAPE is : {mean_absolute_percentage_error(y_true, y_pred)}')
    print(f'R2 is : {metrics.r2_score(y_true, y_pred)}',end='\n\n')
    
    result = {}
    result['MSE'] = metrics.mean_squared_error(y_true, y_pred)
    result['MAE'] = metrics.mean_absolute_error(y_true, y_pred)
    result['RMSE'] = np.sqrt(metrics.mean_squared_error(y_true, y_pred))
    result['MAPE'] = mean_absolute_percentage_error(y_true, y_pred)
    result['R2'] = metrics.r2_score(y_true, y_pred) 
    
    return(result)

Baseline-approach: Multivariate Time Series – VAR¶

A Multivariate time series has more than one time-dependent variable. Each variable depends not only on its past values but also has some dependency on other variables. Vector Auto Regression (VAR) is one of the most commonly used method for multivariate time series prediction. So here VAR was selected to model the baseline performance of predicting pressure given predictors in past time steps.

For VAR models, Akaike and Bayesian Information Criterion are two ways of scoring a model based on its log-likelihood and complexity, the less the better, so here both AIC and BIC were used to select lag order of the VAR mdoel. A lag is the value of a variable in a previous time period, so a pth-order VAR refers to a VAR model which includes lags for the last p time periods. The study tested lag order from 1 to 9 and selected the best lag order to build the VAR model from training dataset.

Deep Learning Models¶

Recent developments in deep learning methods offers a lot of promise for time series modeling. It can learn the temporal dependence and handle temporal structures like trends and seasonality without manual feature curation. Most used deep learning networks in time-series analysis include Recurrent Neural Network (RNN), Long short-term memory (LSTM), and Gated Recurrent Units (GRU), and Convolutional Neural Network (CNN). This study explored Bi-LSTM, GRU, and CNN.

To model the time-series problem, there are # important parameters that we need to define first: (1) history window: the time window within which the data is used as predictors (2) Single-step time-series forecasting vs Horizontal-stype: Single-step is a technique where the model is exposed to one history window of data at a time, such as days, weeks, months, quarters, or years, and attempts to predict the next consecutive step; Horizon-style time-series forecasting is a technique where the model is exposed to one window of data at a time, such as days, weeks, months, quarters, or years, and attempts to predict the next n consecutive steps based on selection

def custom_ts_multi_data_prep(dataset, target, start, end, window, horizon):
    X = []
    y = []
    start = start + window
    if end is None:
        end = len(dataset) - horizon

    for i in range(start, end):
        indices = range(i-window, i)
        X.append(dataset[indices])

        indicey = range(i+1, i+1+horizon)
        y.append(target[indicey])
    return np.array(X), np.array(y)

from tqdm import tqdm
def custom_ts_multi_data_prep(dataset, target, start, end, window, horizon):
    X = []
    y = []
    start = start + window
    if end is None:
        end = len(dataset) - horizon

    for i in range(start, end, horizon):
        indices = range(i-window, i)
        X.append(dataset[indices])

        indicey = range(i+1, i+1+horizon)
        y.append(target[indicey])
    return np.array(X), np.array(y)
